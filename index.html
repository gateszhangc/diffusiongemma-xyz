How It Works

The diffusion process is built on a Gemma 2 text encoder and a learned denoising U-Net

One text encoder, one denoising backbone, and a step-by-step reverse diffusion process keep the system focused enough to understand and powerful enough to generate high-quality images.

01

Encode the text prompt

The Gemma 2 text encoder processes the input prompt and produces a rich embedding that guides the diffusion process toward the described content.

02

Start from random noise

The denoising U-Net begins with a random noise tensor and iteratively refines it over a fixed number of steps, guided by the text embedding at each timestep.

03

Iterative denoising

Each step of the reverse diffusion process removes a small amount of noise. The U-Net predicts the noise residual, conditioned on the text embedding and current timestep.

04

Generate the final image

After the full denoising schedule, the latent representation is decoded into a high-resolution image that reflects the original text prompt.

Why It Matters

It brings open diffusion to the Gemma ecosystem.

Most open diffusion models rely on CLIP or T5 for text encoding. DiffusionGemma replaces them with Gemma 2, creating a unified open-weight ecosystem from language understanding to image generation.

Quick Start

How to go from reading about DiffusionGemma to generating images

The model requires a GPU with at least 16GB VRAM, Python 3.10+, and the Hugging Face diffusers library. If that matches your setup, this is the shortest path to a first generation.

01

Install dependencies

Install the required packages including diffusers, transformers, and torch with CUDA support.

pip install diffusers transformers torch accelerate
02

Load the model

Download and load the DiffusionGemma model and its Gemma 2 text encoder from Hugging Face.

from diffusers import DiffusionPipeline pipe = DiffusionPipeline.from_pretrained( "google/diffusion-gemma" )
03

Generate an image

Run inference with a text prompt. The model returns a PIL image after the denoising steps complete.

image = pipe( "a serene mountain lake at sunset", num_inference_steps=50 ).images[0]
04

Experiment with parameters

Adjust guidance scale, inference steps, and seed to explore the model's capabilities and refine outputs.

image = pipe( prompt, guidance_scale=7.5, num_inference_steps=100, generator=torch.manual_seed(42) ).images[0]
FAQ

The fastest answers to the questions people ask first

Start here if you want the creator, the architecture, or the hardware requirements without reading the full paper first.

Who created DiffusionGemma?
DiffusionGemma comes from Google DeepMind. The model was announced in May 2025 and builds on the Gemma 2 language model family to create an open diffusion model for text-to-image generation.
What text encoder does DiffusionGemma use?
DiffusionGemma uses a Gemma 2 language model as its text encoder, replacing the CLIP or T5 encoders used by most other open diffusion models. This choice creates a unified ecosystem across Google's open-weight language and image models.
What hardware do you need to run it?
The model requires a GPU with at least 16GB of VRAM for reasonable inference speeds. Consumer GPUs like the NVIDIA RTX 4090 or A-series workstation cards are suitable. CPU inference is possible but significantly slower.
Is DiffusionGemma available for commercial use?
DiffusionGemma is released under the Gemma license, which permits most research and commercial applications. Review the official license on the model card for the specific terms and conditions.
